# 宠伴管家：宠物门店预约会员管理系统需求文档

## 1. 项目概述

### 1.1 项目名称

宠伴管家

### 1.2 项目类型

宠物门店预约会员管理系统。

### 1.3 项目副标题

面向宠物店、宠物洗护门店的预约、订单、会员、客户与宠物档案管理系统。

### 1.4 项目定位

本项目的核心是宠物门店业务管理系统，主要解决宠物门店在预约、订单、会员、客户和宠物档案管理中的效率问题。

AI Agent 不是系统的必需功能，而是增强模块。即使不启用 AI，系统也应能够完整完成用户预约、商家确认、订单管理、会员管理和基础数据统计等核心业务流程。

## 2. 项目背景

许多宠物店日常需要处理大量重复工作，例如客户咨询价格、预约洗护、确认到店时间、管理会员卡、记录宠物信息、提醒客户复购等。

传统门店通常依赖微信手动沟通，容易出现回复慢、预约冲突、客户记录混乱、会员消费不清晰等问题。

本项目希望打造一个面向宠物门店的小程序和后台管理系统，让客户可以自助预约宠物洗护、美容、寄养等服务，商家可以在后台管理预约、订单、客户、宠物档案和会员卡。AI Agent 作为后续增强能力，用于辅助客服回复、经营分析、客户召回和营销文案生成。

## 3. 目标用户

### 3.1 商家用户

商家用户包括宠物店老板、店长、前台和宠物美容师。

典型需求：

- 管理洗护、美容、寄养等服务项目。
- 查看每日预约情况。
- 管理客户和宠物档案。
- 管理会员卡、套餐卡和充值记录。
- 分析门店收入和服务热度。
- 维护客户关系，提高客户复购率。

### 3.2 C 端用户

C 端用户主要是养宠用户。

典型需求：

- 查看门店服务项目和价格。
- 给宠物预约洗护、美容、寄养等服务。
- 查询订单和预约状态。
- 查看会员卡余额或套餐次数。
- 咨询服务价格、注意事项和可预约时间。
- 接收预约提醒和活动通知。

## 4. 核心业务价值

### 4.1 对客户的价值

- 预约更方便。
- 服务信息更清楚。
- 可以管理自己的宠物档案。
- 可以查看会员卡和套餐卡状态。

### 4.2 对商家的价值

- 减少重复沟通。
- 降低预约冲突概率。
- 提高客户和宠物档案管理效率。
- 提升会员管理和客户复购能力。
- 通过数据统计辅助经营决策。

### 4.3 对项目展示的价值

- 能体现完整业务系统设计能力。
- 能体现前端、后端、数据库、权限和部署能力。
- 能体现真实业务建模能力。
- 后续可通过 AI Agent 展示工具调用、知识库问答和数据分析能力。

## 6. AI Agent 增强模块

### 6.1 AI 模块定位

AI Agent 是锦上添花功能，不是 MVP 的必要前提。

系统应优先保证核心业务闭环稳定运行，再逐步接入 AI 能力。AI 模块主要用于提升项目亮点和用户体验。

### 6.2 智能客服 Agent

面向用户端。

能力：

- 回答服务项目、价格、营业时间、地址和预约规则。
- 根据用户描述推荐服务。
- 查询可预约时间。
- 引导用户选择宠物、服务和时间。
- 对复杂问题提示转人工。

示例问题：

```text
我家 8 斤的猫洗澡多少钱？
明天下午还有位置吗？
狗狗美容需要多久？
第一次来需要注意什么？
猫咪很怕吹风可以洗吗？
```

可调用工具：

```ts
queryKnowledgeBase()
getServiceList()
getServicePrice()
getAvailableTimeSlots()
createBookingDraft()
createManualSupportTicket()
```

### 6.3 经营分析 Agent

面向商家后台。

能力：

- 查询收入数据。
- 查询预约趋势。
- 分析热门服务。
- 分析客户复购情况。
- 给出经营建议。

示例问题：

```text
这个月收入怎么样？
最近哪个服务最受欢迎？
本周预约比上周多还是少？
猫咪客户和狗狗客户哪个更多？
哪些服务利润比较高？
```

可调用工具：

```ts
getRevenueStats()
getBookingStats()
getPopularServices()
getCustomerRepeatRate()
getMemberConsumptionStats()
```

### 6.4 客户召回 Agent

面向商家后台。

能力：

- 找出 30/60/90 天未消费客户。
- 根据客户历史消费生成召回策略。
- 生成微信私聊文案。
- 生成优惠券草案。
- 创建跟进任务。

示例问题：

```text
帮我找出 60 天没来的客户。
给猫咪洗护客户生成一批召回文案。
哪些客户快流失了？
帮我设计一个老客户回店活动。
```

可调用工具：

```ts
getChurnRiskCustomers()
getCustomerProfile()
getCustomerPetProfiles()
createCouponDraft()
generateRecallMessage()
createFollowUpTask()
```

### 6.5 营销文案 Agent

面向商家后台。

能力：

- 生成朋友圈文案。
- 生成微信群活动文案。
- 生成小红书风格文案。
- 生成节假日活动方案。
- 根据空闲时段生成促销建议。

示例问题：

```text
帮我写一条周末猫咪洗护活动文案。
今天下午预约比较少，帮我做个临时促销。
春节寄养服务怎么宣传？
帮我写一条小红书风格的宠物美容文案。
```

可调用工具：

```ts
getIdleTimeSlots()
getServiceList()
createPromotionPlan()
generateMarketingCopy()
```

## 15. 项目亮点包装

### 15.1 项目简介

宠伴管家是一个面向宠物洗护门店的预约会员管理系统，包含用户端预约、宠物档案、商家后台、会员卡、订单管理和数据统计等功能。系统支持宠物门店完成从用户预约、商家确认、服务完成到会员消费记录的完整业务闭环。

AI Agent 作为可选增强模块，支持智能客服、经营分析、客户召回和营销文案生成。Agent 可调用业务工具查询数据库、分析预约和消费数据，并生成可执行的经营建议。

### 15.2 技术亮点

- 使用 uni-app + Vue 3 构建用户端小程序。
- 使用 Vue 3 + Element Plus 构建商家后台。
- 使用 NestJS + Prisma 构建业务 API。
- 使用 PostgreSQL 进行核心业务数据存储。
- 支持预约冲突检测。
- 支持客户、宠物、订单、会员卡等核心业务建模。
- 支持后台数据看板。
- 后续可使用 PostgreSQL + pgvector 实现知识库检索。
- 后续可使用 AI Agent 工具调用实现业务数据查询和经营分析。
