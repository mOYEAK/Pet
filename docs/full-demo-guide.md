# 宠伴管家 Demo 全方位演示文档

本文档用于完整演示「宠伴管家」项目，覆盖本地启动、Docker/Nginx 部署、演示账号、业务主流程、AI 亮点、验收命令和常见排错。

## 1. 演示目标

本 Demo 要证明系统已经具备一个宠物门店 MVP 的完整能力：

- C 端用户可以查看门店、服务、宠物档案、预约、会员、套餐卡和智能客服。
- 商家后台可以管理预约、服务、客户、宠物、订单、会员、套餐卡、知识库和系统设置。
- 业务闭环可以跑通：用户预约 -> 后台确认 -> 生成订单 -> 优惠券核销/支付/套餐卡核销 -> 消费记录。
- 站内通知可以提醒用户预约提交、预约确认、预约取消和订单支付完成。
- AI 增强可以演示：用户智能客服、后台经营分析、客户召回、营销文案。
- 项目具备交付包装：演示数据、文档、Docker Compose、Nginx 网关配置。

## 3. 本地启动命令

推荐日常演示优先使用本地开发模式，启动速度快，便于看日志。

### 3.1 安装依赖

```bash
pnpm install
```

### 3.2 启动数据库

```bash
docker compose up -d postgres redis
```

查看容器状态：

```bash
docker compose ps
```

### 3.3 初始化数据库

```bash
pnpm --filter @petcare/api prisma:generate
pnpm --filter @petcare/api prisma:migrate
pnpm --filter @petcare/api prisma:seed
```

如果需要重置演示数据：

```bash
docker compose down -v
docker compose up -d postgres redis
pnpm --filter @petcare/api prisma:migrate
pnpm --filter @petcare/api prisma:seed
```

### 3.4 启动三端服务

建议开三个终端分别运行：

```bash
pnpm dev:api
```

```bash
pnpm dev:admin
```

```bash
pnpm dev:miniapp
```

也可以一次性启动：

```bash
pnpm dev
```

## 4. Docker/Nginx 部署命令

项目已提供 Docker Compose 编排：

- `postgres`: PostgreSQL 数据库
- `redis`: Redis 预留缓存
- `api`: NestJS API 服务
- `web`: Nginx 静态站点和反向代理

### 4.1 构建镜像

```bash
pnpm docker:build
```

等价于：

```bash
docker compose --profile app build
```

### 4.2 启动完整部署

```bash
pnpm docker:up
```

等价于：

```bash
docker compose --profile app up -d
```

### 4.3 查看日志

```bash
docker compose logs -f api
docker compose logs -f web
```

### 4.4 停止服务

```bash
pnpm docker:down
```

等价于：

```bash
docker compose --profile app down
```

### 4.5 清空数据库卷

谨慎使用，会删除演示数据：

```bash
docker compose down -v
```

### 4.6 Docker 镜像拉取问题

如果 `docker build` 长时间无输出，或出现 Docker Hub token/EOF 错误，通常是基础镜像拉取网络问题。可以先手动拉取：

```bash
docker pull node:20-bookworm-slim
docker pull node:20-alpine
docker pull nginx:1.27-alpine
```

如仍失败，优先切换 Docker 镜像源或登录 Docker Hub。现场演示时建议使用本地开发模式，避免受网络影响。

## 6. 推荐演示流程

### 6.1 开场介绍

建议讲解词：

> 这是一个面向宠物洗护/美容单门店的预约会员管理系统。它包含 C 端 H5/小程序用户体验、商家后台、NestJS API、Prisma/PostgreSQL 数据库，以及可降级的 AI 工具调用能力。核心目标是跑通从用户预约到后台处理、订单支付、会员消费记录的完整业务闭环。

重点说明：

- 当前定位是 MVP/课程项目/作品集项目。
- 真实支付、短信通知、多门店 SaaS、复杂 RBAC 和向量检索是后续扩展。
- 当前重点是业务建模完整、三端联动真实、演示链路稳定。

### 6.2 C 端用户体验

打开 C 端 H5：

```text
http://localhost:5174
```

演示内容：

1. 首页
   - 展示门店名称、营业时间、地址、电话。
   - 展示预约规则和活动入口。
   - 说明用户可以从首页快速进入服务和预约。

2. 服务页
   - 展示猫咪洗护、小型犬洗护、宠物美容护理等服务。
   - 演示宠物类型/体型筛选。
   - 点击服务查看详情、价格、耗时、注意事项。

3. 宠物档案
   - 展示演示宠物“咪咪”。
   - 说明用户可以维护宠物类型、品种、年龄、体重、备注等信息。

4. 创建预约
   - 选择宠物。
   - 选择服务。
   - 选择到店日期。
   - 选择固定时间段，例如 `10:30 - 12:00`。
   - 提交预约。

5. 我的预约
   - 查看预约状态。
   - 查看订单状态。
   - 演示待确认/已确认预约可以取消。

6. 会员中心
   - 展示会员等级、余额、积分。
   - 展示套餐卡剩余次数和有效期。
   - 展示消费记录。

7. 智能客服
   - 点击快捷问题或手动输入：
     - `猫咪洗护多少钱？`
     - `明天下午还有位置吗？`
     - `门店营业时间和地址是什么？`
     - `套餐卡怎么使用？`
     - `最近有什么活动？`
     - `帮咪咪预约明天下午3点的猫咪洗护`
   - 查看客服根据宠物、服务和实时档期生成的预约草案。
   - 点击“去预约”，确认预约页已回填，但仍需用户主动提交。

8. 通知中心
   - 从首页点击“通知”入口。
   - 查看预约提交、预约确认和订单支付提醒。
   - 演示单条标记已读和全部已读。

讲解重点：

- C 端不是静态页面，所有核心数据来自 API。
- 预约时间使用固定门店时间段。
- 会员和套餐卡是后续支付闭环的用户侧展示入口。

### 6.3 商家后台基础能力

打开商家后台：

```text
http://localhost:5173
```

使用管理员手机号登录：

```text
19900000000
```

演示内容：

1. 工作台
   - 今日预约数。
   - 待确认预约。
   - 今日收入。
   - 热门服务。

2. 服务管理
   - 查看服务列表。
   - 说明支持服务启停、新增、编辑。
   - 展示服务价格、宠物类型、体型、耗时、注意事项。

3. 预约管理
   - 查看 C 端刚创建的预约。
   - 修改预约状态为已确认。
   - 从预约生成订单。

4. 预约日历
   - 按日期和固定时间段展示预约。
   - 说明门店可以用来安排排班、发现时间冲突。

5. 订单管理
   - 查看从预约生成的订单。
   - 演示一种或多种支付方式：
     - 到店支付：`STORE_PAY`
     - 会员余额：`MEMBER_BALANCE`
     - 模拟支付：`MOCK_PAY`
     - 套餐卡核销：`PACKAGE_CARD`
   - 非套餐卡支付时可选择客户未使用且满足门槛的优惠券，系统自动计算优惠金额和实收金额。

6. 会员管理
   - 查看会员余额、积分。
   - 查看套餐卡次数。
   - 查看客户优惠券状态。
   - 查看消费记录。

讲解重点：

- 后台可以承接 C 端预约。
- 订单支付成功后，订单状态会变为已支付，预约会变为已完成。
- 优惠券核销会记录优惠金额，并把客户券状态变为已使用。
- 会员余额支付会扣余额、加积分、写消费记录。
- 套餐卡支付会扣次数、写消费记录。
- 预约状态变化和订单支付完成会写入 C 端站内通知。

### 6.5 AI 增强能力

演示内容：

1. 知识库管理
   - 展示预约规则、护理注意事项、门店说明。
   - 说明 C 端智能客服可以引用这些内容回答问题。

2. AI 经营助手
   - 输入：
     - `本月收入怎么样？`

   - 展示系统基于订单、预约、热门服务生成经营分析。

3. 客户召回
   - 选择 30/60/90 天未消费客户。
   - 生成召回文案。
   - 创建跟进任务。
   - 修改跟进任务状态。

4. 营销文案
   - 输入活动目标，例如：
     - `周末猫咪洗护活动`
     - `下午空闲时段促销`
     - `老客户回店活动`
   - 生成朋友圈、微信群、小红书风格文案。
   - 展示优惠活动草案，并可一键创建“满 128 减 20”优惠券模板。

5. 优惠券管理
   - 打开 `http://localhost:5173/coupons`。
   - 创建或编辑满减券模板。
   - 将优惠券发放给演示客户 `18800000000`。
   - 回到订单管理，在非套餐卡支付方式下选择优惠券完成核销。
   - 回到 C 端会员中心，查看优惠券状态从未使用变为已使用。

6. 通知记录
   - 打开 `http://localhost:5173/notifications`。
   - 查看预约提交、预约确认、取消和支付通知。
   - 进入客户详情，查看该客户最近通知。

讲解重点：

- C 端客服支持 OpenAI 兼容模型工具调用；未配置密钥、超时或模型失败时自动使用规则回答。
- AI 的价值来自业务数据调用，而不是单纯聊天。
- 当前知识库仍使用关键词检索，后续可以接入 pgvector、embedding 和 RAG。

## 8. 可直接使用的讲解词

### 开场

> 宠伴管家是一个面向宠物洗护门店的预约会员管理系统。用户可以在 C 端查看服务、维护宠物档案并预约到店时间；商家可以在后台处理预约、生成订单、管理客户和会员；系统还提供智能客服、经营分析、客户召回和营销文案能力。

### 业务闭环

> 这里我演示的是从客户预约到商家完成收款的完整闭环。C 端提交预约后，后台可以确认预约并生成订单。订单支持到店支付、会员余额支付、套餐卡核销和模拟支付。支付完成后，订单状态、预约状态、会员余额、套餐卡次数和消费记录都会同步更新。

### AI 模块

> AI 模块不是孤立聊天窗口，而是围绕业务数据做增强。用户侧客服可以查询服务、价格、营业时间和预约时间；商家侧经营助手可以读取订单、预约和热门服务数据；客户召回可以基于长期未消费客户生成跟进文案；营销助手可以根据服务和空闲时间段生成活动文案。

### 项目边界

> 当前版本定位是单门店 MVP，所以没有接真实微信支付、短信通知、多门店 SaaS 和复杂权限。AI 客服支持可选真实模型并保留规则降级，后续可以继续扩展真实支付、微信订阅消息、RBAC 和 pgvector 知识库检索。

## 9. 常见问题与排错

### 登录或页面请求返回 500

检查 API 和数据库：

```bash
docker compose ps
pnpm dev:api
```

查看 API 日志：

```bash
docker compose logs -f postgres
```

### 页面没有数据

重新 seed：

```bash
pnpm --filter @petcare/api prisma:seed
```

### 端口被占用

检查端口：

```bash
netstat -ano | findstr :3000
netstat -ano | findstr :5173
netstat -ano | findstr :5174
netstat -ano | findstr :8080
```

### Docker 构建失败

先验证 Docker 是否正常：

```bash
docker version
docker compose ps
docker pull node:20-alpine
```

如果是 Docker Hub 网络问题，切换镜像源后重试：

```bash
pnpm docker:build
pnpm docker:up
```

### 后台 Docker 模式刷新 404

确认访问：

```text
http://localhost:8080/admin/
```

不要访问：

```text
http://localhost:8080/admin
```
